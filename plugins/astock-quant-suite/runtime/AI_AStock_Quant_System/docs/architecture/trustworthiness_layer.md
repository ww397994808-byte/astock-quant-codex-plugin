# Trustworthiness Layer

## 1. 为什么回测能跑不等于可信

回测能跑，只说明代码没有崩。可信回测还需要数据正确、交易规则真实、费用完整、没有未来函数、样本外不过度退化，并且能解释为什么结果可以进入下一阶段。

## 2. 数据质量的重要性

错误数据会制造虚假收益。V4 增加 Data Quality Layer，检查日期重复/逆序/缺失、非交易日、价格区间错误、成交量异常、异常跳变和疑似复权问题。

## 3. Readiness 分类体系

- `INVALID`：审计失败、未来函数 HIGH、T+1 或交易规则违规。
- `RESEARCH_ONLY`：审计通过但样本外差、稳定性差或交易次数不足。
- `PAPER_READY`：审计通过、样本外正常、稳定性正常、交易次数足够。
- `LIVE_CANDIDATE`：在 PAPER_READY 基础上，还需要长区间、多周期、多标的和风险达标。

## 4. 压力测试意义

压力测试用连续涨停、连续跌停、停牌、极端震荡、零成交量、异常跳空、单边牛市和单边熊市检验策略是否能在极端场景下保持规则可执行。

## 5. 实盘前必须通过哪些检查

实盘前至少必须通过：

- 数据质量检查；
- 未来函数审计；
- A股交易规则审计；
- Readiness 不为 INVALID；
- QMT dry_run / enable_real_trade / 二次确认；
- pre-trade check；
- emergency_stop 未触发。
